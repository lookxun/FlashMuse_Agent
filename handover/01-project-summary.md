# Project Summary

## 项目目标

做一个内部使用的“简单版即梦”，核心体验是：

1. 直接进入主页，不登录
2. 用户在聊天框里输入需求
3. 系统理解上下文
4. 系统自动优化提示词
5. 可以生成图片或视频
6. 保存历史记录

当前阶段：

- 先把核心流程跑通
- 先给内部人员测试
- 后面再补登录、支付、正式推广能力

## 当前技术栈

- 框架：`Next.js 16`
- 前端：`React 19`
- 样式：`Tailwind CSS 4`
- 包管理：`npm`

20. 图片模式显示模型占位、比例、分辨率、风格参数
21. 视频模式显示模型占位、比例、分辨率、风格、时长参数

26. 当前默认视频模型为 OpenRouter 视频模型 `google/veo-3.1-lite`
27. AI 回复支持基础富文本排版、标题、分隔线、红 / 蓝提示底色块

30. 不同历史会话可并发生成，发送按钮只锁当前会话
31. AI 新回复支持 1-8 秒自适应逐字打字机效果，旧历史回复不重新打字
32. 左侧历史会话和“映造正在思考”支持运行中光环扩散动画
33. Agent 模式下明确生图 / 生视频请求会自动路由到图片 / 视频生成流程
34. 新增 `/api/intent`，通过 OpenRouter 做 AI 意图分类，配合硬规则和本地纠错记忆减少图 / 视频误判
35. 输入框支持上传和粘贴图片预览，最多 5 张，超过时显示黑色提示框
36. AI 回复底部新增反馈操作：复制、重新生成、喜欢、不喜欢、回答不对、模式错了、更多菜单
37. 反馈会写入本地日志 `yinzao-feedback-log-v1`，模式错了会写入本地纠错记忆 `yinzao-intent-memory-v1`
38. 反馈区“更多”菜单支持复制文字和删除整条 AI 回复
39. 上传图片会进入对话流，并随请求传给 Agent / 图片 / 视频接口
40. Agent 可基于上传图片做图片理解；图片模式可参考图生图；视频模式可按上传图或最近图片做首帧 / 视觉参考
41. Agent 自动路由策略已收紧，图片分析类表达不再误触发生图
42. 视频生成已从 Seedance 独立聚合接口切到 OpenRouter 视频接口：创建 `POST /api/v1/videos`，查询 `GET /api/v1/videos/{jobId}`
43. Seedance 独立聚合接口先暂停，后续部署到服务器后再继续处理白名单和专线接入
44. 发送交互已修复：发送后立即显示用户消息、清空输入框、按钮显示 `发送中...`，并加同会话防重复发送
45. Agent 简单问候和简单追问已收紧，不再随便输出多段、多条“下一步方向”
46. 左侧栏 UI 已微调：Logo 为紫底白图标，“Yinzao”改“映造”，“AI 创作助手”改“AI影片助手”，“开启新对话”改“新建对话”
47. 视频模式当前强制 4 秒、720p，提示词太短会强制扩写成完整视频提示词，前 2 分钟每 10 秒轮询，之后每 30 秒轮询，超过约 5 分钟提示重试
48. OpenRouter 视频查询和下载已增加 `curl` 兜底，解决 Node `fetch` 查询视频任务偶发假 404 和内容下载鉴权问题
49. 图片生成已修复 `modalities`，默认图片模型 `bytedance-seed/seedream-4.5` 只请求图片输出
50. 图片 / 视频生成中状态卡统一为 400x400 动画卡片，带生成中百分比、状态文案和已等待计时
51. 视频生成完成后会直接在对话流里显示视频卡片，不再只显示打开按钮；视频按比例缩小，鼠标悬停自动播放，移开暂停
52. 当前选中的历史对话会保存到 `yinzao-active-session-v1`，刷新浏览器后继续停留在该对话
53. Agent 模式在意图识别阶段会立即显示“映造正在思考”，避免发送后无反馈；识别为图片 / 视频后切换成对应生成等待卡片
